Badge

<mh-badge> is a small status pill — a building block for table cells, record headers, and board cards. It's an atom, not where MaxHTML competes: it exists so composites have a pill to drop in.

Example

Colour rides on the tone attribute

Status colour is carried by a native tone attribute — success / warning / danger / info, with no attribute meaning neutral. The attribute draws a leading colour dot and a faint same-hue tint from the shared [tone] axis; the label stays as text (it's domain-specific — "Lead", "Customer", "Churned" — so the stylesheet can't supply it). This replaced the old leading-emoji dot: tone is themeable (override the four --mh-* tokens), recolorable, and ejects cleanly to a variant prop. Pipeline stages with no good/bad valence (Lead, New, Customer's plain classification) stay neutral.

Count / decorative badges use an <mh-icon>

When a badge carries a glyph + data rather than a status (a comment count, a promo tag), drop an <mh-icon> inside instead of a tone:

Usage

<mh-badge tone="success">Customer</mh-badge>
<td>…<mh-badge tone="danger">Churned</mh-badge>…</td>     <!-- in a table cell -->
<h1>Ada Lovelace <mh-badge tone="success">Customer</mh-badge></h1>  <!-- record header -->
<mh-badge><mh-icon name="message"></mh-icon> 18</mh-badge>   <!-- count badge -->
