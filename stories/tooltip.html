Tooltip (CSS-only)

<mh-tooltip> wraps a trigger and its tip. The tip is the child with role="tooltip" — a native, self-describing slot, not sibling position. It appears on hover or keyboard focus and floats a fixed offset above the trigger. No JS: the float is chrome (a constant offset from its own trigger), so it lives in the stylesheet — a tooltip looks like the canvas but isn't, because its position is fixed by the component, never per-instance data.

Example — hover or tab to it

Accessibility is free and JS-free

Put aria-describedby="id" on the trigger and a matching id on the <small role="tooltip">. Keyboard users get it via :focus-within; screen readers get it via the description. All native attributes — no class, no data-*, no script.

For a plain hint, skip the component

A bare text hint doesn't need any of this — use the native title attribute: <button title="Delete forever"><mh-icon name="trash"></mh-icon></button>. It's the type=button of tooltips: content on a native attribute, zero vocabulary. Reach for <mh-tooltip> only when you want it themed, on focus, or richer.

Placement is automatic — the model emits none of it

This is the token rule in action: a tooltip's side is a runtime layout decision (it depends on where the trigger renders — something the model can't see), so the model never emits it. There is no placement attribute to type. It prefers top, and where CSS anchor positioning is supported the browser auto-flips it below near the viewport's top edge — zero emitted placement, zero JS. Older browsers degrade to a static top tooltip.

The one real limit (graduation path)

A tip inside an overflow:hidden / scrolling ancestor still clips, and if you need shift (slide along the edge) rather than just flip, that's the mh-menu-style graduation: a small mh-tooltip.js that drops in Floating UI (the Popper successor) to position the tip in the top layer. The kit still only styles the tip; Floating UI does the geometry. Crucially this changes nothing the model emits — placement stays out of the markup whether it's done by anchor-positioning CSS or by the script.

Usage

<mh-tooltip>
  <button aria-describedby="t1"><mh-icon name="trash"></mh-icon> Delete</button>
  <small role="tooltip" id="t1">Delete forever</small>
</mh-tooltip>
