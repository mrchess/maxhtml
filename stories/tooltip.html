Tooltip (CSS-only)

<mh-tooltip> wraps a trigger and its tip. The tip is the child with role="tooltip" — a native, self-describing slot, not sibling position. It appears on hover or keyboard focus and floats a fixed offset above the trigger. No JS: the float is chrome (a constant offset from its own trigger), so it lives in the stylesheet — a tooltip looks like the canvas but isn't, because its position is fixed by the component, never per-instance data.

Example — hover or tab to it

Accessibility is free and JS-free

Put aria-describedby="id" on the trigger and a matching id on the <small role="tooltip">. Keyboard users get it via :focus-within; screen readers get it via the description. All native attributes — no class, no data-*, no script.

For a plain hint, skip the component

A bare text hint doesn't need any of this — use the native title attribute: <button title="Delete forever">🗑</button>. It's the type=button of tooltips: content on a native attribute, zero vocabulary. Reach for <mh-tooltip> only when you want it themed, on focus, or richer.

Limits (and the graduation path)

Default placement is top — no manual placement hook (that'd be an open-ended variant the kit avoids). A tip inside an overflow:hidden ancestor will clip, and edge-collision flipping needs CSS anchor positioning or a small mh-tooltip.js behavior layer — the same graduation path as mh-menu. Until then it's zero JS, zero data hooks.

Usage

<mh-tooltip>
  <button aria-describedby="t1">🗑 Delete</button>
  <small role="tooltip" id="t1">Delete forever</small>
</mh-tooltip>
