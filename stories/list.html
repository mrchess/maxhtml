List

A vertical list of rows. Each <mh-item> has a leading <mh-avatar>, a <header> (name + meta), and a trailing <footer> of actions. The key move: open-ended actions go in an overflow <mh-menu>, so "I might want more actions later" costs nothing and the row never gets crowded.

Example

Anatomy

Primary action vs secondary is self-describing (<button> vs <button type=button>) — never by position. The kebab menu is a slot-scoped <mh-menu>: ghost button, no caret.

Usage

<mh-list>
  <mh-item>
    <mh-avatar>AL</mh-avatar>
    <header><strong>Ada Lovelace</strong><small>12 mutual friends</small></header>
    <footer>
      <button type=button>Message</button>
      <mh-menu><button>⋯</button><a><mh-icon name="poke"></mh-icon> Poke</a><a><mh-icon name="user-minus"></mh-icon> Remove</a></mh-menu>
    </footer>
  </mh-item>
</mh-list>
