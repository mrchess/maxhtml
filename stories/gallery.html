Gallery

<mh-gallery> is an image-dominant tile grid — the unit of both an album browser and a photo grid. Each tile is a <figure> with an <img> cropped square to fill the cell, plus an optional <figcaption> (title + meta). Caption present = album cover; absent = a bare photo. This is what mh-card (padded, text-first) and mh-grid (no image cropping) can't do: an edge-to-edge image as the hero of the tile.

Album browser — tiles with captions

Photo grid — bare tiles, each opens a lightbox

Drop the <figcaption> and the same composite is a dense photo grid. Wrap each <img> in a <button commandfor="id"> to open the lightbox below.

Lightbox — a <dialog> with an image

The photo viewer isn't a new tag: a native <dialog> whose content is an <img> styles itself as a wide, dark lightbox. It reuses mh-dialog.js verbatim — open with commandfor, close on the backdrop, Esc, or a footer button. The variant is keyed on structure (the dialog has an image), never a class.

Usage

<mh-gallery>
  <figure>                                  <!-- album cover: captioned -->
    <a href="/album/iceland"><img src="…" alt="Iceland"></a>
    <figcaption><strong>Iceland 2026</strong><small>248 photos</small></figcaption>
  </figure>
  <figure><button commandfor="lb"><img src="…" alt="Photo"></button></figure>  <!-- bare photo -->
</mh-gallery>

<dialog id="lb">                              <!-- lightbox = dialog + img -->
  <img src="…" alt="Glacier lagoon">
  <p>Jökulsárlón · 28 Jun 2026</p>
  <footer><button type=button><mh-icon name="download"></mh-icon> Download</button><button>Close</button></footer>
</dialog>
<script src="mh-dialog.js"></script>
