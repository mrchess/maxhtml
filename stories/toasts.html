Toasts

<mh-toasts> is a fixed, viewport-anchored stack of transient notifications (bottom-right). Children are reused <mh-alert>s — severity still rides the leading emoji. The finding: a toast's position is chrome — a constant viewport offset, like a tooltip's anchor — so it lives in the stylesheet, not a per-element data hook, and not a wall. It renders and stacks with zero JS. Only the lifecycle (spawn on an event, auto-dismiss) is behavior — a progressive-enhancement layer (mh-toast.js), exactly like mh-menu/mh-dialog.

Try it (mh-toast.js loaded)

Usage

<!-- static: renders + positions + stacks with ZERO JS -->
<mh-toasts>
  <mh-alert>✅ <strong>Deploy finished.</strong> v2.3.1 is live.<button type=button>✕</button></mh-alert>
</mh-toasts>

<!-- behavior (mh-toast.js): auto-dismiss + spawn on an event -->
<button data-toast="✅ <strong>Saved.</strong> …">Save</button>   <!-- declarative -->
mhToast('⚠️ <strong>High latency.</strong> api-eu p95 > 500ms');     // programmatic
