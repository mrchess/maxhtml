Charts

Charts are data-driven geometry — the same class of thing that walls the canvas (see Whiteboard). They split along that boundary: a mark that's a single scalar (a bar height, a donut slice) rides a narrow custom-property data hook — the sanctioned pattern from mh-note's --x/--y. A mark that's multi-point geometry (a line/area path) can't be a scalar, so it needs a raw <svg> of coordinates — the full escape hatch. Either way the frame is a reused mh-card and all colour/stroke/sizing lives in the stylesheet.

Bar chart — <mh-bars> + <mh-bar style="--v:.62">

--v is data (0–1); the bar's height is calc(var(--v) * 100%) in the sheet.

Donut — <mh-donut style="--p1:40; --p2:30; …">

Segment percentages are data on custom props; the conic-gradient is assembled in the sheet from the categorical palette (--mh-c1…c5). Legend swatches are emoji dots that match the palette.

Line / area — a styled inline <svg> (escape hatch)

Multi-point geometry can't be a scalar, so the points live in the markup. The kit styles by SVG element type (no classes): polygon = area fill, polyline = line, circle = points, line = gridlines. Use preserveAspectRatio="none" so it fills the card.

The honest read

Bars and donut sit right on the data-hook line — one number per mark, carried on a custom property, with every visual rule still in the sheet. The line is past it: coordinate geometry can only live inline. So a dashboard is mostly on-rails chrome (shell, stat tiles, table, the mh-card frames) with charts as data-geometry islands — exactly the boundary the saturation experiment maps.
