Themes

A theme is just a block of --mh-* token overrides, applied by setting data-theme on <html> — a host concern, never emitted by the UI. Click one and the whole explorer (chrome + the sample below) re-skins live; your choice persists.

How it works

Every visual value in the kit comes from a token, and every rule is zero-specificity (:where()), so a theme overrides cleanly — no specificity fight, no touched markup. A dark theme would also flip --mh-hover to a translucent white so hovers stay visible (see the prompt in themes.css).

Add your own

Drop a block into themes.css (the file header has a copy-paste prompt to generate one from any palette):

:root[data-theme="midnight"] {
  --mh-bg: #0f1115; --mh-surface: #161a20; --mh-fg: #e8eaed;
  --mh-muted: #9aa4af; --mh-line: #262b33;
  --mh-accent: #5aa6ff; --mh-accent-fg: #07101f;
  --mh-hover: rgba(255,255,255,.08); --mh-radius: 12px;
}

Note: the full-page recipe previews are separate documents (their own <iframe>, linking only maxhtml.css), so they stay on the default theme — only this explorer re-skins.
