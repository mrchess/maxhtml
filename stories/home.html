The HTML kit built for LLMs.

A closed vocabulary of composite layouts and page recipes. The model writes markup only — one amortized stylesheet handles the design.
1 · Ready-made building blocks

A closed kit of pieces — from a button up to a whole dashboard shell. These big layouts are exactly what plain CSS and widget libraries leave you to hand-build.
2 · The styling is loaded once

The kit's design lives in one CSS file. The AI writes only tag names like <mh-card> — never colors or spacing. The tag says what it is; the file knows how it looks.
3 · Consistent by default

The AI reaches for a ready-made recipe first. What it won't do is hand-roll raw HTML and CSS — that's the drift and token drain the kit exists to prevent.

<script src="https://mrchess.github.io/maxhtml/maxhtml.js"></script>
<body>
  <!-- paste the model's MaxHTML markup here -->
</body>

2 · Paste this system prompt into Claude, then ask for any screen.

You build UI as MaxHTML — a closed kit of mh-* components styled by an already-loaded stylesheet. Emit MARKUP ONLY: no style="", no <style>/CSS, no <div>/<span>, no class/data-*. Variants ride native attributes (<button> = primary, <button type=button> = secondary, tone="danger" …). Need a component you don't know? Ask — don't improvise.
