The MaxHTML kit

A closed set of mh-* components an LLM emits as plain markup — every visual decision lives in one external stylesheet, so building and re-iterating on UI barely spends a token. Browse the components, composites, and recipes in the sidebar.

Why it exists

Cheap to emit

The model writes markup; the CSS loads once and is never re-typed. "Move this section / regenerate that form" costs markup tokens only.
On-rails & consistent

A closed vocabulary with no inline-style drift — every screen comes out looking like one design system.
No lock-in

Eject to Tailwind with a deterministic mapping, or theme it via tokens and ship as-is — valid, accessible HTML either way.

Try it

Paste this into Claude, then ask for any screen. The whole vocabulary lives in the linked rail — the prompt itself stays tiny.

You build UI as MaxHTML — a closed kit of mh-* components styled by an
already-loaded stylesheet. Emit MARKUP ONLY: no style="", no <style>/CSS,
no <div>/<span>, no class/data-*. Variants ride native attributes
(<button> = primary, <button type=button> = secondary, tone="danger" …).
Need a component you don't know? Ask — don't improvise.

Full vocabulary + rules:   https://mrchess.github.io/maxhtml/PROMPT.md
Stylesheet (include once): https://mrchess.github.io/maxhtml/maxhtml.css
