UI that barely costs a token.

An opinionated component kit so an LLM can generate and restyle whole interfaces cheaply — it writes markup, the styling stays out of the prompt.

Why MaxHTML

Cheap to emit

~40% fewer tokens than a hand-rolled dashboard. The model writes markup; the CSS lives outside and never gets re-typed.
On-rails & consistent

A closed vocabulary means no inline-style drift. Every screen comes out looking like the same design system.
No lock-in

Eject to Tailwind with a deterministic mapping, or theme it and ship as-is — valid, accessible HTML either way.

Try it

Paste this into Claude, then ask for any screen. The whole vocabulary lives in the linked rail — the prompt itself stays tiny.

You build UI as MaxHTML — a closed kit of mh-* components styled by an
already-loaded stylesheet. Emit MARKUP ONLY: no style="", no <style>/CSS,
no <div>/<span>, no class/data-*. Variants ride native attributes
(<button> = primary, <button type=button> = secondary, tone="danger" …).
Need a component you don't know? Ask — don't improvise.

Full vocabulary + rules:   https://mrchess.github.io/maxhtml/PROMPT.md
Stylesheet (include once): https://mrchess.github.io/maxhtml/maxhtml.css
