The HTML kit built for LLMs.

A closed vocabulary of composite layouts and page recipes. The model writes markup only — one amortized stylesheet handles the design.

How it works

"Put the styling in one file" isn't the whole story — plenty of CSS kits do that. The real trick is a fixed set of building blocks, plus a rulebook that keeps the AI using them.

1 · Ready-made building blocks

A closed kit of pieces — from a button up to a whole dashboard shell or a fill-in-the-blank page recipe. Those big layouts are exactly what plain CSS and widget libraries leave you to hand-build.
2 · The styling is loaded once

The kit's design lives in one CSS file the browser loads a single time. The AI writes only tag names like <mh-card> — never colors, spacing, or shadows. The tag says what it is; the file already knows how it looks.
3 · Consistent by default

The AI reaches for a ready-made recipe first, then snaps the blocks together freely when none fits exactly. What it won't do is hand-roll raw HTML and CSS — that's the drift and the token drain the kit exists to prevent.

Try it

Paste into Claude, then ask for any screen.

You build UI as MaxHTML — a closed kit of mh-* components styled by an
already-loaded stylesheet. Emit MARKUP ONLY: no style="", no <style>/CSS,
no <div>/<span>, no class/data-*. Variants ride native attributes
(<button> = primary, <button type=button> = secondary, tone="danger" …).
Need a component you don't know? Ask — don't improvise.

Full vocabulary + rules:   https://mrchess.github.io/maxhtml/PROMPT.md
Stylesheet (include once): https://mrchess.github.io/maxhtml/maxhtml.css
