Message

<mh-message> is one comment in a forum thread or one message in a chat stream: a leading <mh-avatar>, a <header> (author <strong> + <small> timestamp), the body (one or more <p>), and an optional <footer> of quiet actions. Nest a <mh-message> inside another for threaded replies — they indent under their parent with a thread line. It's the rich, multi-line, nestable cousin of mh-item (which is a centred single-line row, right for flat lists, not conversations).

Thread — nested replies

Chat — flat stream with reactions

Anatomy

The avatar sits top-left; author, body, actions, and any nested replies stack beside it (a 2-column grid, no wrapper element). Footer actions are quiet text buttons in a thread (Reply, ▲ votes) or mh-badge reactions in chat. The flat thread list (topic + author + reply count) is a separate screen — that's just mh-list/mh-item, reused, not this.

Usage

<mh-message>
  <mh-avatar>AL</mh-avatar>
  <header><strong>Ada Lovelace</strong><small>· 3h ago</small></header>
  <p>Does anyone still hand-write CSS in 2026?</p>
  <footer><button type=button>↰ Reply</button><button type=button>▲ 24</button></footer>

  <mh-message>                          <!-- nested = threaded reply -->
    <mh-avatar>AT</mh-avatar>
    <header><strong>Alan Turing</strong><small>· 2h ago</small></header>
    <p>Mostly tokens + a closed component set now.</p>
  </mh-message>
</mh-message>
