Menu (behavior layer)

A dropdown — the first component that needs JavaScript. CSS can fake a hover/focus reveal, but it can't close on outside click, close on Escape, or expose aria-expanded state. So mh-menu is a real Web Component, loaded via <script src="maxhtml.js">.

Example — try it

What the JS adds

Click the trigger to toggle; click outside or press Esc to close. The button carries aria-haspopup and a live aria-expanded. The JS also restructures the DOM — it wraps the authored <a> items in a generated <menu> popup, something CSS cannot do.

Progressive enhancement

The authored markup is just a <button> followed by <a> links. With the script loaded it's a dropdown; with no JS it degrades to a visible button and a plain list of links — still usable.

Usage

<mh-menu>
  <button>Account</button>
  <a href="/profile">Profile</a>
  <a href="/billing">Billing</a>
  <a href="/signout">Sign out</a>
</mh-menu>

<script src="maxhtml.js"></script>
