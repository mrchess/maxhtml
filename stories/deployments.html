Deployments (recipe)

A cloud-hosting deployments page (think Vercel/Railway) — and a zero-new-composite result. The whole screen is reuse: the shell (mh-app/mh-navbar/mh-sidemenu), the mh-page header with a primary Deploy button, and three mh-cards. The deployment "table" is a mh-list/mh-item: branch + status mh-badge + commit/author/time in the item <header>, and the row actions (Promote / Rollback / Delete) in the item <footer> — the per-row action footer is an item shape, not a tabular grid, so a real <table> would have been the wrong fit. Primary vs secondary rides native attributes (<button> vs <button type=button>); the active row's Promote is just disabled. Env vars are a plain <table> in a card with an Add variable footer button, and the Danger zone is a card with a tone="danger" mh-alert over its actions.
