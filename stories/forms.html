Form controls

Native controls the kit styles (it doesn't reinvent atoms). A switch is a checkbox with role="switch" — native and self-describing, no class, no new tag. A field hint is a <small> inside the <label>; an error rides the native aria-invalid attribute. A <label> that wraps a checkbox/radio flips to an inline row automatically (keyed on structure).

Example

Usage

<label>Email
  <input type=email value="…">
  <small>Hint text</small>          <!-- field description -->
</label>
<input type=email aria-invalid="true">  <!-- error state, native attr -->

<select><option>…</option></select>

<label><input type=checkbox> Agree</label>       <!-- inline row -->
<label><input type=radio name=plan> Monthly</label>
<label><input type=checkbox role=switch> On</label>  <!-- toggle -->

A switch beside a setting name + description is a <mh-list> row: name in the item <header>, switch in the item <footer> — see the Settings recipe.
