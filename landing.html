UI that barely costs a token.

An opinionated component kit so an LLM can generate and restyle whole interfaces cheaply — it writes markup, the styling stays out of the prompt.

Why MaxHTML

Cheap to emit

~40% fewer tokens than a hand-rolled dashboard. The model writes markup; the CSS lives outside and never gets re-typed.
On-rails & consistent

A closed vocabulary means no inline-style drift. Every screen comes out looking like the same design system.
No lock-in

Eject to Tailwind with a deterministic mapping, or theme it and ship as-is — valid, accessible HTML either way.

Try it

Paste this into Claude, then ask for any screen. It builds in MaxHTML and barely spends a token on styling.

You build UI as MaxHTML — a closed set of components styled by an external
stylesheet that is already loaded. Emit MARKUP ONLY.

Rules
1. Output only HTML markup. Never write style="...". Never output <style> or CSS.
2. Use only the tags below. Never use <div>, <span>, or any other primitive.
   Never invent components. Never add class or data-* attributes.
3. Variants come from tags/native attributes only: <button> = primary,
   <button type=button> = secondary.
4. If you need something not listed, ask — don't improvise.

Composite layouts
  <mh-app>      dashboard shell: <mh-navbar> + <mh-sidemenu> + <main>
  <mh-layout>   two columns: <mh-sidemenu> + <main>
  <mh-page>     a <header> (h1 + action buttons) then the page body
  <mh-grid>     responsive grid of tiles / cards
  <mh-hero>     centered hero: <h1> + <p> + <footer> of buttons
  <mh-section>  centered band: <h2> + body + optional <footer>

Components
  <mh-sidemenu> brand <header> + <nav> of <mh-submenu> groups
  <mh-submenu>  <header> label + <a> links (active link: aria-current="page")
  <mh-navbar>   <strong> brand + <a> links
  <mh-card>     <header> + body + <footer> actions
  <mh-stat>     <header> label + <strong> value + <small> delta
  <mh-menu>     <button> trigger + <a> items
  <table>       plain native table; the kit styles it

Use these semantic tags directly: main, nav, header, footer, h1-h6, p, a,
form, label, input, button, strong, small.

Include once in <head>:
  <link rel="stylesheet" href="https://mrchess.github.io/maxhtml/maxhtml.css">

Now build: a dashboard with a sidebar, four stat tiles, and a table of
recent activity.
